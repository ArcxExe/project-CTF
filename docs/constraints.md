# CTF Platform - Архитектурные ограничения (Constraints)

## Технологический стек
- **Язык:** Java 21
- **Фреймворк:** Spring Boot 3
- **База данных:** PostgreSQL
- **Миграции БД:** Flyway
- **Сборка:** Gradle
- **Контейнеризация:** Docker, Docker Compose

## Правила написания кода (Code Guidelines)
1. **Строгая типизация:** Использование `var` допускается только там, где тип очевиден из правой части выражения.
2. **REST API Контракты:** Использование DTO (Data Transfer Objects) для всех входящих и исходящих запросов. Запрещено возвращать сущности БД напрямую (Entity Leak).
3. **Безопасность:**
   - JWT для аутентификации.
   - Пароли хранятся в зашифрованном виде (BCrypt).
   - Строгое разграничение доступов на уровне контроллеров (Role-based Access Control).
4. **Обработка ошибок:** Глобальный `@ControllerAdvice` для формирования стандартизированных JSON ответов об ошибках (RFC 7807 Problem Detail или кастомный формат).
5. **Тестирование:** 
   - Unit-тесты для сервисного слоя (JUnit 5 + Mockito).
   - Integration-тесты для репозиториев (Testcontainers).
6. **Документация API:** Интеграция Swagger / OpenAPI 3.
7. **Код-стайл:** Следование стандартам Java (CamelCase, понятные имена переменных, отсутствие "магических чисел").
