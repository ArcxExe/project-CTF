# Детальный план реализации проекта (CTF Platform & Academic Grading)

**Статус готовности MVP:**
- **Backend:** ~70% (Миграции БД, сущности, базовые контроллеры присутствуют, но есть критические дыры в безопасности).
- **Frontend:** ~60% (Инфраструктура FSD готова, API-сервисы написаны, но UI сильно завязан на mock-данные и локальные стейты, отсутствует реактивность).

Настоящий документ описывает пошаговый план доработки и стабилизации платформы. План составлен на основе аудита кодовой базы и сопоставления с `docs/requirements.md`, `docs/constraints.md` и `docs/architecture.md`.

---

## 1. Базовые настройки и Безопасность (Security)
- [x] Миграции БД (Flyway V1 - V10) созданы и соответствуют требованиям.
- [x] Настроена базовая аутентификация через JWT и пароли (BCrypt).
- [ ] ⚠️ (Требует доработки: Изоляция роли `STUDENT`)
  - Контроллеры `AcademicGroupController` и `AcademicFlowController` содержат мутирующие эндпоинты (`POST`, `PATCH`, `DELETE`), не защищенные аннотацией `@PreAuthorize("hasRole('ADMIN')")`. Любой студент может удалять учебные потоки.
- [ ] ⚠️ (Требует доработки: Настройка SecurityFilterChain)
  - Не все административные контроллеры подпадают под маску `/api/admin/**`. Необходимо привести URL к единому стандарту или добавить аннотации проверки ролей на уровне методов для всех `Admin*Controller`.
- [ ] ⚠️ (Требует доработки: Отказ от Entity Leak)
  - `QuizController` возвращает сущности `QuizSubmission`, `QuizQuestion`, `QuizOption` напрямую. Требуется внедрить DTO для этих эндпоинтов.
